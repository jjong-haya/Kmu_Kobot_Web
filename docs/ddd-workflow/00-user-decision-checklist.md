# 00. 사용자 결정 체크리스트

## 1. 목적

### 1.1 왜 먼저 결정하는가

KOBOT Web은 인증, 가입 승인, 공식 팀, 프로젝트 팀, 초대, 투표, 연락 요청, 감사 로그, 개인정보가 서로 연결된 운영 시스템입니다.

구현 전에 사용자가 정해야 할 것과 우리가 안전하게 기본값으로 둘 수 있는 것을 분리해야 합니다.

### 1.2 구현 차단 규칙

| 우선순위 | 의미 | 처리 규칙 |
| --- | --- | --- |
| P0 | 틀리면 권한, 개인정보, 승인 흐름이 깨짐 | 결정 전 구현 금지 또는 명시적 기본값 필요 |
| P1 | 구현은 가능하지만 나중에 변경 비용 큼 | 권장 기본값으로 진행 가능, 문서에 ASSUMPTION 기록 |
| P2 | UI/문구/편의성 수준 | 합리적 기본값으로 진행 가능 |

## 4. 새 질문 처리 규칙

### 4.1 질문 생성 규칙

작업 중 다음 중 하나라도 나오면 이 파일에 질문을 추가한다.

- 권한 주체가 새로 등장한다.
- 공개/비공개 정보가 바뀐다.
- 상태 전이가 추가된다.
- 개인정보 필드가 추가된다.
- 외부 연동이 추가된다.
- 감사 로그 payload가 바뀐다.

### 4.2 질문 문장 형식

```text
왜 [행위자]가 [범위]에서 [행위]를 할 수 있어야 하는가?
틀리면 어떤 사고가 나는가?
```
