# Public 영역 분석

## 1. PublicLayout의 역할

`src/app/layouts/PublicLayout.tsx`는 얇은 shell이다.

포함 요소

- `ScrollToTop`
- `Outlet`
- footer

빠진 요소

- 공통 헤더
- 공통 상단 내비게이션
- 공통 브레드크럼
- 공통 데이터 로더

결론적으로 public 영역은 "하나의 통일된 사이트 shell"보다 "페이지들이 footer만 공유하는 모음"에 가깝다.

## 2. 공통 패턴

public 페이지들은 다음 조합을 반복한다.

- shadcn 스타일 카드, 배지, 입력창, 버튼
- 파일 내부 정적 데이터
- 검색과 필터를 위한 로컬 state
- 빈 상태 카드
- 실제 서버 동작 없이 끝나는 목업 액션

공통 문제도 분명하다.

- 날짜가 하드코딩되어 금방 낡는다.
- 같은 성격의 데이터가 페이지별로 중복된다.
- 상단 내비게이션 경험이 페이지마다 다르다.

## 3. 페이지별 성격

### Landing

파일: `src/app/pages/public/Landing.tsx`

역할

- 공개 홈페이지이자 핵심 홍보 랜딩

주요 섹션

- 고정 헤더
- 히어로와 CTA
- 기술 태그
- 캘린더와 일정 팝오버
- about 벤토 그리드
- 트랙 소개와 커리큘럼 모달
- 업데이트와 이벤트 카드
- 플레이북
- 갤러리 프리뷰
- FAQ 프리뷰
- 최종 모집 CTA

강점

- 상호작용 밀도가 가장 높다.
- 브랜드의 첫인상을 만드는 역할을 충분히 수행한다.

문제

- 파일이 너무 크다.
- FAQ 검색과 카테고리 필터 일부가 실동작보다 장식에 가깝다.
- 링크 데이터와 실제 section id가 어긋나는 부분이 있다.
- public 화면에서 member 전용 경로로 직접 보내는 링크가 섞여 있다.
- `2026-03-25` 기준으로 모집 관련 날짜와 D-day가 이미 낡았다.

### Activities

파일: `src/app/pages/public/Activities.tsx`

역할

- 활동 아카이브와 갤러리

주요 섹션

- 홈 복귀 헤더
- 검색
- 태그 필터
- 정렬
- 월별 masonry 그리드
- 데스크톱용 월 내비게이션
- 라이트박스

강점

- public 페이지 중 인터랙션 설계가 좋다.
- 정보량이 많아도 브라우징 경험을 고려했다.

문제

- 외부 링크 대부분이 `#`다.
- 날짜 정렬이 브라우저 의존 `Date` 파싱에 기대고 있다.
- 라이트박스가 body scroll을 직접 잠그며 unmount cleanup 리스크가 있다.
- 실제 데이터 소스가 없어서 유지보수 비용이 크다.

### FAQ

파일: `src/app/pages/public/FAQ.tsx`

역할

- 자주 묻는 질문 전용 페이지

구성

- 복귀 헤더
- 검색창
- 카테고리 필터
- 결과 수 표시
- 아코디언
- 추가 문의 CTA

평가

- 구조는 명확하다.
- 데이터가 전부 하드코딩이라 Landing의 FAQ와 쉽게 어긋난다.

### Projects

파일: `src/app/pages/public/Projects.tsx`

역할

- 공개 프로젝트 카탈로그

구성

- 검색
- 상태 필터
- 카드 그리드
- empty state

평가

- 전형적인 목록형 페이지다.
- 구조는 안정적이지만 실제 프로젝트 상세 진입이나 데이터 연결은 없다.

### Notice

파일: `src/app/pages/public/Notice.tsx`

역할

- 공개 공지 목록

구성

- 검색
- 카테고리 필터
- 고정 공지
- 일반 공지

평가

- 게시판형 IA는 잡혀 있다.
- 실제 CMS나 상세 페이지로 이어지는 도메인 모델은 없다.

### Recruit

파일: `src/app/pages/public/Recruit.tsx`

역할

- 모집 소개와 지원 폼

구성

- 모집 상태 배지
- D-day
- 핵심 정보 카드
- 타임라인
- 모집 트랙
- 지원 폼

문제

- 제출은 `toast.success`만 실행한다.
- 실제 폼 유효성 검증과 전송이 없다.
- 날짜 정보가 이미 낡았다.

### Contact

파일: `src/app/pages/public/Contact.tsx`

역할

- 연락처와 문의

구성

- 연락처 카드
- 소셜 링크
- 지도 placeholder
- 문의 폼
- 하단 미니 FAQ

문제

- 지도는 placeholder다.
- 문의 폼은 실제 전송이 없다.

## 4. Public 영역 총평

public 영역은 "디자인 데모와 홍보 자료"로서는 충분히 풍부하다. 하지만 운영 가능한 사이트로 보려면 다음이 반드시 필요하다.

- 공통 헤더와 navigation 통합
- 콘텐츠 데이터의 중앙화
- 날짜/상태의 실시간화
- 폼 전송 처리
- broken link와 member 경계 정리
